# Phase 1 — Database Schema & Core Types

**Goal:** Define all Convex tables, indexes, search indices, and TypeScript types.

## Tables

### `users`
Linked to Clerk via `tokenIdentifier`. Every signed-up user gets a row.

| Field | Type | Description |
|-------|------|-------------|
| `tokenIdentifier` | `string` | Clerk's canonical stable user ID |
| `name` | `string` | Full name |
| `email` | `string` | Email address |
| `avatarUrl` | `optional string` | Profile picture URL |
| `role` | `union("seeker", "employer")` | User type |
| `phone` | `optional string` | Contact phone |
| `location` | `optional string` | City/region |
| `bio` | `optional string` | Short summary for seekers |
| `resumeStorageId` | `optional Id("_storage")` | Uploaded resume file |
| `skills` | `optional array(string)` | Skills list for seekers |
| `companyId` | `optional Id("companies")` | If role=employer, their company |
| `onboarded` | `boolean` | Whether profile setup is complete |

**Indexes:**
- `by_token_identifier` — `["tokenIdentifier"]` (unique lookup)

### `companies`
Employer company profiles.

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | Company name |
| `slug` | `string` | URL-friendly name |
| `description` | `string` | About the company |
| `logoStorageId` | `optional Id("_storage")` | Company logo |
| `website` | `optional string` | Company website |
| `location` | `optional string` | Headquarters |
| `size` | `optional string` | e.g. "1-10", "11-50", "51-200" |
| `industry` | `optional string` | Industry sector |
| `ownerId` | `Id("users")` | User who created the company |

**Indexes:**
- `by_slug` — `["slug"]` (unique)
- `by_owner` — `["ownerId"]`

### `jobs`
Job listings posted by employers.

| Field | Type | Description |
|-------|------|-------------|
| `companyId` | `Id("companies")` | Posting company |
| `title` | `string` | Job title |
| `description` | `string` | Full job description |
| `requirements` | `optional string` | Requirements section |
| `responsibilities` | `optional string` | Responsibilities section |
| `location` | `string` | Job location |
| `locationType` | `union("on-site", "remote", "hybrid")` | Work type |
| `employmentType` | `union("full-time", "part-time", "contract", "temporary", "internship")` | Employment type |
| `salaryMin` | `optional number` | Minimum salary |
| `salaryMax` | `optional number` | Maximum salary |
| `salaryCurrency` | `optional string` | Currency code (default "USD") |
| `category` | `optional string` | Job category slug |
| `skills` | `optional array(string)` | Required skills |
| `status` | `union("active", "closed", "draft")` | Listing status |
| `applicationCount` | `number` | Denormalized counter |
| `featured` | `optional boolean` | Featured/promoted listing |

**Indexes:**
- `by_company` — `["companyId"]`
- `by_status` — `["status"]`
- `by_status_and_created` — `["status", "_creationTime"]`
- `by_company_and_status` — `["companyId", "status"]`
- `by_category_and_status` — `["category", "status"]`

**Search index:**
- `search_title_description` — search fields: `["title", "description"]`

### `applications`
Job applications from seekers.

| Field | Type | Description |
|-------|------|-------------|
| `jobId` | `Id("jobs")` | Job being applied to |
| `userId` | `Id("users")` | Applicant |
| `resumeStorageId` | `optional Id("_storage")` | Submitted resume |
| `coverLetter` | `optional string` | Cover letter text |
| `status` | `union("pending", "reviewing", "shortlisted", "rejected", "hired")` | Application status |
| `employerNotes` | `optional string` | Private notes for employer |

**Indexes:**
- `by_job` — `["jobId"]`
- `by_user` — `["userId"]`
- `by_job_and_status` — `["jobId", "status"]`
- `by_user_and_job` — `["userId", "jobId"]` (prevents duplicates)

### `saved_jobs`
Bookmarked jobs for seekers.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | `Id("users")` | Seeker |
| `jobId` | `Id("jobs")` | Saved job |

**Indexes:**
- `by_user` — `["userId"]`
- `by_user_and_job` — `["userId", "jobId"]` (unique)

### `categories`
Job categories / industries.

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | Display name |
| `slug` | `string` | URL slug |
| `icon` | `optional string` | Icon identifier |

**Indexes:**
- `by_slug` — `["slug"]` (unique)
